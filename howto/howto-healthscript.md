# Howto: (Re)create the Health Check Script

## Background

The health check script is generated by the provisioning script and placed at:

```
/home/<username>/ubuntu_health_check.sh
```

It provides a colour-coded status report covering:

- SSH configuration
- Swap and disk usage
- Memory usage
- TCP transmit offloading status
- Failed systemd units
- Recent failed SSH logins
- Pending package updates
- Service status (running/stopped/failed/not installed) for all installed services
- AppArmor profile status per service
- Recent AppArmor denials
- IP blocklist status (entry count and last update time)
- Port usage with UFW allow/block status
- Smoke tests — HTTP checks for Nginx, Grafana, Forgejo, Monit, Webmin;
  database connection tests for MySQL/MariaDB, PostgreSQL, Valkey;
  Mosquitto pub/sub test

The script is regenerated from scratch on each provisioning run, so it always
reflects the currently installed services.

---

## Conf file

```bash
cat > /tmp/healthscript.conf << 'EOF'
INSTALL_DEFAULT=n
USER_CREATE_SUDO_USER=n
USER_USE_CURRENT_USERNAME=y
CREATE_HEALTHSCRIPT=y
EOF
```

Run:

```bash
sudo bash ubuntu_provision.sh /tmp/healthscript.conf
```

---

## Run the health check

```bash
sudo ~/ubuntu_health_check.sh
```

Or without sudo if the sudoers entry is configured:

```bash
~/ubuntu_health_check.sh
```

---

## What the sudoers entry provides

The provisioning script adds a sudoers entry allowing the sudo user to run
the health check without a password:

```
username ALL=(ALL) NOPASSWD:/home/username/ubuntu_health_check.sh
```

This is written to `/etc/sudoers.d/<username>-commands` alongside other
NOPASSWD commands like `apt-get update` and `systemctl`.

---

## Verify

**Check the script exists and is executable:**

```bash
ls -la ~/ubuntu_health_check.sh
```

**Run and check output:**

```bash
sudo ~/ubuntu_health_check.sh 2>&1 | head -40
```

**Check the sudoers entry:**

```bash
sudo cat /etc/sudoers.d/<username>-commands
```

---

## Notes

- The health check script is regenerated on every provisioning run that has
  `CREATE_HEALTHSCRIPT=y`. Any manual edits to the script will be overwritten.
- The smoke tests use `curl -4` to force IPv4, avoiding IPv6 connection issues
  that can occur on Ubuntu 26.04 when services listen only on `127.0.0.1`.
- MySQL and MariaDB connection tests use a passwordless `healthcheck@localhost`
  user created during database installation. No root credentials are stored in
  the health check script.
- The script must be run as root (or via sudo) since several checks require
  elevated privileges (AppArmor status, UFW status, ipset list).
